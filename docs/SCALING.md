# 대규모 심사 운영 가이드 — 모델 티어링·비용·배치

200~300개 기업(현재 테스트 배치의 20~40배)을 심사할 때의 비용 구조, 모델 역할 분담,
소요 시간, 배치 API 도입 판단 기준. (2026-07-06 작성, 요금은 작성 시점 추정 —
실행 전 [Gemini 요금표](https://ai.google.dev/pricing) 재확인 권장)

## 모델 티어링 (현행 구현)

비싼 모델(pro)은 "추론이 필요한 곳"에만, 나머지는 flash. 이미 코드에 반영돼 있다.

| 단계 | 모델 | 호출 수 (300기업 기준) | 왜 이 모델인가 |
|---|---|---|---|
| 문서 변환 (`batch_convert`) | 없음 (soffice) | 0 | API 불필요 |
| 파일 심사 1차 (`screen_batch`) | **flash** | ~3,300 (기업당 ~11파일) | 서명·날인 시각판정 + 필드 추출. 문서당 독립이라 저렴한 모델로 전량 처리 |
| 에스컬레이션 (`screen_batch`) | **pro** | ~165 (5% 내외) | uncertain·confidence<0.7·서명 '불명확' 건만 재심사. `ESCALATION_CONF` 로 조절 |
| 교차검증 (`cross_check`) | **pro** | 300 (제출건당 1회) | 다문서 추론(대표자↔서명자, 국가 경보, 자격 규칙 계산) — 텍스트만 보내므로 호출당 토큰 작음 |
| 재실행 | — | 0 | 같은 파일+같은 프롬프트 버전이면 캐시 히트, 비용 0 |

**pro를 쓰지 않는 곳**: 1차 파일 심사 전체(95%), 문서 변환, 필수서류 코드레벨 체크
(파일명 매칭, API 0회), 엑셀 생성.

## 비용 추정 (300기업 신규 1회전)

가정: 기업당 11파일, 파일당 평균 7페이지(≈258tok/p), flash $0.30/$2.50,
pro $1.25/$10.00 (1M 토큰 입력/출력, 2.5 세대 기준).

| 단계 | 계산 | 추정 비용 |
|---|---|---|
| flash 1차 (3,300건) | (3.2k in + 0.8k out)/건 ≈ $0.003 | **~$10** |
| pro 에스컬레이션 (165건) | (3.2k in + 0.8k out)/건 ≈ $0.012 | **~$2** |
| pro 교차검증 (300건) | (6k in + 1k out)/건 ≈ $0.018 | **~$5** |
| **합계** | | **~$17–20 / 회전** |

민감도: 에스컬레이션 비율이 5%→15%로 늘면 +$4. 페이지 수가 2배면 flash 비용 ~2배.
프롬프트 버전을 올리면 캐시가 무효화되어 전량 재호출된다 — **버전 변경은 배치 시작 전에**.

## 소요 시간 추정

| 단계 | 순차 | 권장 워커 | 병렬 시 |
|---|---|---|---|
| 변환 (soffice, CPU) | ~3h | 4 (로컬 CPU) | ~45분 |
| flash 1차 (~12s/건) | ~11h | 8–12 | **~1–1.5h** |
| pro 교차검증 (~35s/건) | ~3h | 4 | **~45분** |
| **전체 (신규)** | ~17h | | **~3h** |

- 워커 수 근거: flash 는 유료 티어 RPM 여유가 크고(수백~수천 RPM), pro 는 RPM 이 낮아
  4 워커 권장. 429 가 반복되면 워커를 줄인다 (자동 백오프는 구현돼 있음).
- 실행: `screen_batch.py <dir> 10` / `cross_check.py <json> 4` (둘째 인자가 워커 수).
- 중단돼도 재실행하면 캐시 히트로 이어서 진행된다 (체크포인트 역할).

## 배치 API (Gemini Batch Mode) — 도입 판단

Gemini 배치 모드는 **비용 50% 할인**, 대신 완료 목표가 24시간(보통 그보다 빠름)인
비동기 방식이다. JSONL 로 요청을 묶어 제출하고 폴링해서 수집한다.

**기대효과 (300기업 기준)**
- flash 1차를 배치로: $10 → $5. 전체 $17 → **~$12 (약 30% 절감)**
- 시간은 절감이 아니라 **교환**: 대화형 1.5h → 최대 24h 대기 (보통 수 시간)

**권고: 지금은 도입 보류, 조건 충족 시 도입**
- 현재 회전당 비용이 ~$17 수준이라 절감액($5)보다 파이프라인 비동기화(제출→폴링→수집,
  에스컬레이션·캐시 재배선) 구현·운영 비용이 크다.
- 도입이 맞아지는 조건: ① 심사가 정기 반복 업무가 되어 연간 회전 수가 늘 때
  ② 기업 수가 1,000+ 로 늘 때 ③ 마감 여유가 하루 이상일 때.
- 도입 시 flash 1차만 배치로 보내고, 에스컬레이션·교차검증은 결과 수집 후
  실시간 pro 로 유지하는 2단 구성이 적합하다.

## 신뢰성 (20배 규모에서 견디는 것들)

- 네트워크 예외(타임아웃·연결끊김)도 재시도: flash 300s, pro 600s 타임아웃 + 3회 백오프
- 파일 1개 실패가 배치를 죽이지 않음 (needs_review 로 격리)
- 교차검증 3회 실패 시 해당 제출건만 uncertain 처리하고 계속 진행
- 429 는 Retry-After 헤더 준수
